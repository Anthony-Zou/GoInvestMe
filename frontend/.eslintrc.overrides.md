# ESLint Configuration Override

This file contains ESLint rule overrides for specific patterns that are intentional in our codebase.

## Disabled Rules

### react-hooks/set-state-in-effect
**File**: `src/components/ConnectWallet.tsx`  
**Reason**: Legitimate use of `setState` in `useEffect` for SSR/client hydration handling. This is the recommended Next.js pattern for avoiding hydration mismatches.

### @typescript-eslint/no-explicit-any
**Files**: `src/lib/logger-utils.ts`, `src/lib/sentry-utils.ts`, `src/__tests__/*.ts`  
**Reason**: Utility libraries require `any` types for maximum flexibility when accepting user-provided contexts, error objects, and test mocks.

### @typescript-eslint/no-require-imports
**Files**: `src/__tests__/*.ts`, `src/lib/logger.ts`  
**Reason**: CommonJS `require()` is needed for:
- Conditional server-side imports (logger.ts)
- Test utilities that don't have ES module exports (test setup files)

## Summary

These are intentional patterns that provide necessary flexibility and functionality. They are properly documented with inline eslint-disable comments.
